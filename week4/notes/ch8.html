Chapter 8: Forms

Forms accept information that can be sent to a backend server using PHP or Ruby. You can also process the information before sending it to the server using JavaScript.

Accessing Form Elements

  • document.forms is a property that returns an HTML collection of all of the forms in the document in the order they appear in the markup.
    • document.forms[0] returns the first form object in the document.
    • document.getElementsByTagname('form') is equivalent to document.forms.
    • The name attribute can be used to identify a form in place of the numerical index. document.forms.search; However if the form has the same name as any properties or methods of the document.forms object then that property or method would be referenced insteaed of the form element. Square bracket notation can be used instead to avoid this. document.forms['search'];
  • A form object has a method called elements that returns an HTML collection of all the elements contained in the form.
    If a form has an input and a button, for example:
    const [input, button] = form.elements;
  • Form controls can be accessed through their names. If the input field's name was searchInput:
    form.searchInput;
    Or square bracket notation:
    form['searchInput'];

Form Properties and Methods

  • form.submit() will submit a form automatically.
  • form.reset() will reset all of the form controls back to their initial values specificed in the HTML. Buttons with the type reset can do this without the need for additional scripting.
  • form.action property can be used to set the action attributed. The URL on the server where the form needs to be sent for procssing is what it should be set to.
    form.action = '/an/other.url'

Submitting a Form

Intercept a submit button by adding an event listener for submit with the following code: 

                const form = document.forms['search'];
                form.addEventListener ('submit', search, false);
                function search() {
                    alert(' Form Submitted');
                }
With this code the form will still be submitted to the url. Use preventDefault() to stop that from happening. Add it to the search function: 
                function search(event) {
                    alert('Form Submitted');
                    event.preventDefault();
                }

Accessing input

Accessing what is in an input takes the following form:
input.value
This can return a value or assignment can be used to put a value in it.
The following code shows how to do hint text for an input by using the focus and blur events: 

                input.addEventListener('focus', function(){
                    if (input.value==='Search Here') {
                        input.value = '' 
                    }
                }, false);
                input.addEventListener('blur', function(){
                    if(input.value === '') {
                        input.value = 'Search Here';
                    } }, false);

Form Controls

  • input fields including text, passwords, check boxes, radio buttons, and file uploads.
    • text input fields: text is the default value for input type. Use type='text' to make the purpose of the field explicit.
    • Password input fields: type='password' is used to enter passwords or secret information. The characters are concealed as soon as they are on screen.
    • Checkbox input fields: type='checkbox' used to select different options that can be checked true or false. User can select more than one checkbox in a list. Boxes can be checked initially using the 'checked' attribute in HTML.
    • Radio button input fields: type='radio' like checkedboxes but only allow one option to be true in a group that has the same Name attribute.
    • Hidden input fields: type='hidden' are not displayed by the browser. Can be used to send information like settings or information that the user has already provided. It will by sent with this form's action attribute. Can be found in HTML, so they should not be used for sensitive information.
    • File input fields: type='file' can be used to upload files. Most broswers provide a browse button or similar that lets a user select a file from their file system.
    • Other input types: HTML5 supports new input types such as number, tel, and color. These will validate automatically, and different browsers will implement different UI elements to let users select these. Currently, browsers that do not support these will use normal text-input fields in their place, so there is no risk to using them.
  • select menus for drop-down lists of options
  • textarea elements for longer text entry
  • button elements for submitting and resetting forms.
    • Submit type to submit a form according to its action attribute.
    • Reset type to clear all form elements. Resets to default values.
    • button type to just be a clickable button that an event listener can be attached to.

Form Validation

Form validation can happen on client side or server side. Both should be used. It is possible to bypass front-end validation and send arbitrary data to the server side, so only secure server side validation should be trusted. Client side validation is good for providing good user experience.
Adding the 'required' attribute in HTML5 will keep a form from being submitted unless the form element has a value selected.
