Functions

Functions can be used to replace code in multiple places that does the same thing. This allows you not to repeat yourself, but also it allows you to update a lot of code just by changing one piece.

  • To invoke a function through a variable you need to place parentheses after it: myFunction(); You won't use the parentheses if you are assigning the function to a new varaible.
  • All functions return a value. The default return value is undefined, but it can be anything put after the return keyword.
  • Arrow Functions
    • Arrow functions are best used for short, anonymous functions.
  • Function Hoisting
    • Hoisting is the javascript interpreter's action of moving all variable and function declarations to the top of the current scope regardless of where they are defined.
    • Functions can be used before they are defined because of hoisting.
    • All function definitions can be placed together, rather than having to define every function before it is used.
  • Variable hoisting
    • variable declarations using the var keyword are hoisted.
    • Assignment is not hoisted. Variables have a value of undefined until assignment.
    • It's best practice to use const or let at the beginninng of a block so hoisting is unnecessary.

Arguments and parameters

Paremeters that a function needs are set when the function is defined, but when the function is invoked it is provided with arguments. Arguments take the place of parameters when they are invoked.

  • Default Parameters can be specified
    • Example:
      function hello(name='World') {
      console.log(`Hello ${name}!); }
    • Default Parameters should always come after non-default parameters because otherwise you would be forced to enter the default parameters anyway.

Callbacks

A function can be passed as an argument to another function. The function that is passed is known as a callback.
Code to check if argument is a function:
if(typeof(callback) === 'function'){
callback();
}

Callbacks are only useful for one-off tasks. It's usually better to keep functions separate and named for reuse.
