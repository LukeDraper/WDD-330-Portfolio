Ch6: Document Object Model Notes

What is the DOM?

The Document Object Model (DOM) represents an html document as a network of connected nodes that form tree-like structure. The DOM treats everything on the web page as a node. The HTML tag is the root node.

The DOM is not part of JavaScript because it is language agnostic, but Javascript is still the language most commonly used with it.

Getting Elements

There are methods to access any element on a page through the DOM. These methods return a nod object or a node list, which is an array-like object. 

                const body = document.body;
Access the document body. This is a Node object with properties and methods we can use to find out more about or modify the body element.

nodeType property can tell us the kind of node. It returns a number. Here are the meanings of some of the numbers:
  • 1 : element
  • 2: attribute
  • 3: text
  • 8: comment
  • 9: body

Legacy DOM Shortcut Methods

  • Document.body returns the body element of a web page.
  • Document.images returns a node list of all the images contained in the document.
  • Document.links returns a node list of all the anchor elements and area element that hava an href attribute.
  • Document.anchors returns a node list of all the anchor elements that have a name attribute.
  • Document.forms returns a node list of all the forms in the document.

Getting Elements

  • document.geteElementById(); This method is probably the most commonly used to access elements on a web page. If no element has the ID provided then null is returned.
  • const listItems = document.getElementsByTagName('li'); This method gets a node list of all elements with the tag named that is provided as an argument.
  • getElementsByClassName() returns a live node list of all elements that have the class name that is supplied as an argument.

Query Selectors

document.queryselector() allows you to use CSS notation to find the first element in the document that matches the CSS selector that is the argument.

document.querySelectorAll() returns a node list of all the elements that matched the query selector.

Navigating the DOM Tree

Some of the nodes generated from HTML are text nodes generated from whitespace characters that show up on page.

  • childNodes property is a list of all the nodes that are children of the node concerned.
  • children property returns any element nodes that are children of that node. It ignores any text nodes.
  • firstChild property returns the first child of a node (might be whitespace text node)
  • lastChild property returns the last child of a node (might return text node from whitespace)
  • nextSibling property returns the next adjacent node of the same parent. Returns null if node is the last Child of that parent.
  • previousSibling property returns the previous adjacent node. Returns null if the node is the first child of that parent.

Finding the Value of a Node

  • nodeValue property returns the text within a text node
  • textContent returns text within a text node. Internet Explorer 8 does not support this, but has the innerText property.

Getting and Setting An Element's Attributes

  • getAttribute() method returns the value of the attribute provided as an argument. Returns null if element does not have attribute.
  • setAttributed method can change the value of an element's attributes. Takes 2 arguments: the attributed and the value that you want that attribute to be.
  • className property allows the class of an element to be set directly.
  • classList property is a list of all classes an element has. Here's its methods to modify the class of an element
    • classList.add() method can add a class to an element without overwriting any existing classes.
    • classList.remove() method removes a specific class from an element
    • classList.toggle() method adds class if the element doesn't have it and removes it if the element does have it. returns true if class is added and false if it is removed.
    • classList.contains() method will check to see if an element has a class

Creating Dynamic Markup

  • document.createElement() method takes a tag name as a parameter and returns that element.
  • document.createTextNode() method takes a parameter, which is a string containing the text that goes in the node.
  • node objects have appendChild() method that adds another node as a child node. You can put a text node made with the createTextNode() method as an argument.
  • The process to add a text node to an element can be shortened using the textContent property 
    const flash = document.createElement('li');
flash.textContent = 'Flash';

Function to Create Elements

It's useful to create functions for creating elements to cut down on the lines of code. 

                function createElement (tag,text) {
                    const el = document.createElement(tag);
                    el.textContent = text;
                    return el
                }

Adding Elements to Page

  • The appendChild() method can add a new child node to the end of the list.
  • insertBefore() places a new element before another element in the markup. This method is called on the parent node and the first parameter is the node to be added. The second parameter is the node it will be added before.

Remove Elements From a Page

  • removeChild() method is called on the parent and removes the child node given as a parameter.

Replacing Elements on a Page

  • replaceChild() method can replace one node with another. Called on parent. Parameters: (new node, node to be replaced)

innerHTML

Returns everything inside an HTML element. All of the raw HTML is returned. innerHTML is also writable. Assignment will replace everything inside. Is very good for inserting large amounts of HTML into the document.

Updating CSS

The style property of each node can be used to dynamically modify the presentation of any element on a web page. CSS properties can be treated like object properties, but they have to be converted to camelCase. For example, Here is how you use the CSS property, background-color:
superman.style.backgroundColor = 'blue';

Bracket notation works too:
superman.style['background color'] = 'blue'; Changing the display property can cause elements to disappeaer and reappear. Set it to 'none' to disappear, and another value to make it reappear.
superman.style.display = 'none';

The style property can be used to check styles except for those set using an external stylesheet, which is the most common way to set styles. The function: getComputedStyle() will return the style of the element passed as an argument. It has methods to get more specific information: 

getComputedStyle(superman).getPropertyCSSValue('color').cssText;


*****Use with Caution *****
It is better practice to dynamically change the class of an element on the fly and keep the styles for each class in a separate stylesheet. There are some cases where it is useful if you don't have access to a stylesheet or its classes.
