Chapter 7: Events

Event Listeners

The blocking approach to event programming has the program check a condition at regular intervals. This blocks the rest of the program from executing.
The non-blocking approach used by Javascript uses event listeners to listen for events to happen. Whenever events happen callback functions are called. The program can execute other code while waiting for the event.

Inline Event Handlers

Inline event handlers are those included directly in markup. Example:

Click Me


                this is p onclick="console.log('You Clicked Me!')">Click Me /p
                with actual opening and closing brackets on the paragraph tags.

Older Event Handlers

Here is an old version of an event hanndler:
document.onclick = function (){ console.log('You clicked on the page!'); }

This method works in almost all browsers, but only one function can be used for each event.

Using Event Listeners

Event listeners are the current standard way to deal with events. Use addEventListener() method. It can be called without a node to be applied to the global object(usually the whole browser window.) 

                document.body.addEventListener('click',doSomething);
The parentheses are not placed after the function when it's used as the argument for the event listener. If they are, then the function will be called when the event listener is set.

When event handlers are triggered the callback function is called, and it automatically passes an event object as a parameter. This can be used to get information about an event. 
                function doSomething(event){
                    console.log(event.type);
                }
Event's properties:
  • Event has a type property that will return the type of event that occurred.
  • the target property returns a reference to the node that fired the event.
  • screenX and screen Y properties show the number of pixels from the left and top of the screen respectively where the event took place.
  • clientxX and clientY properties show the number of pixels from the left and top of the client that is being used(usually the browser window).
  • pageX and pageY properties show the number of pixels from the left and top, respectively, where the event took place in the document. Takes into account whether the page has been scrolled.
